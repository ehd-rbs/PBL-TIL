### 2. 핵심 정리 (내 언어로)

React, TypeScript, Vite 기반 웹 애플리케이션을 구현하고 Vercel을 통해 배포하였다. 로컬에서는 yarn install, yarn dev, yarn build, yarn preview 명령어가 정상 동작하도록 구성하였다.

GitHub 저장소와 Vercel을 연결하여 main 브랜치 push 시 자동 배포되도록 설정하였고, 환경 변수는 코드에 포함하지 않고 Vercel Environment Variables를 통해 관리하였다.

앱에서는 로그인/로그아웃, 데이터 조회/추가/삭제, 페이지 라우팅 기능을 구현하였다. 배포 환경에서도 새로고침과 직접 URL 입력이 정상 동작하는지 확인하였으며, 반응형 레이아웃과 로딩/에러/빈 상태 안내를 적용하였다.

### 4. 느낀 점

이번 과제를 통해 React로 만든 웹 애플리케이션을 실제 배포 환경에서 동작시키는 과정을 경험할 수 있었다. 
특히 로컬에서 잘 실행되는 것과 배포 후 정상적으로 동작하는 것은 다를 수 있다는 점을 알게 되었고, 환경 변수 설정과 라우팅 확인의 중요성을 느꼈다.
GitHub와 Vercel을 연결하여 push만으로 자동 배포가 이루어지는 과정을 보며 실제 서비스 개발 흐름을 조금 더 이해할 수 있었다. 앞으로는 기능 구현뿐만 아니라 빌드, 배포, 사용자 환경에서의 테스트까지 함께 신경 써야겠다고 느꼈다.
